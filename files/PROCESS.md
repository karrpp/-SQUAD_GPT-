# 🏟️ 경기 운영법 (Match Process)

> 프로젝트 진행 시 에이전트 투입 순서와 의사결정 흐름

---

## 경기 흐름 (7단계)

```
1️⃣ 킥오프          → 매니저(성민)가 주제를 던진다
      ↓
2️⃣ 캡틴 판단       → Messi가 상황을 읽고 팀 방향을 잡는다
      ↓
3️⃣ 공격 전개       → Griezmann(간결한 핵심), CR7(임팩트), Son(실행)
      ↓
4️⃣ 미드필드 연결   → Modric(태스크 분해), De Bruyne(사용자 관점), Park(빈틈)
      ↓
5️⃣ 수비 점검       → Ramos(리스크), Cannavaro(구조 검증)
      ↓
6️⃣ 전술 정리       → Coach Guardiola가 종합 → 실행 플랜
      ↓
7️⃣ 최종 세이브     → Buffon이 배포 전 체크
      ↓
⚽ 골!              → 매니저 최종 승인
      ↓
📝 기록             → Zidane이 결정사항 기록
✍️ 하이라이트       → Beckham이 콘텐츠 변환
```

## 상황별 투입 패턴

### 🆕 새 기능 기획 시
```
전원 투입 (풀 스쿼드)
킥오프 → 캡틴 → 공격 → 미드필드 → 수비 → 코치 정리 → GK → 승인
```

### 🐛 버그 수정 시
```
최소 투입
성민 → Son(빠른 진단) → Park(테스트) → Buffon(확인) → 배포
```

### 🎨 UI/UX 개선 시
```
공격 중심
성민 → Messi(방향) → Griezmann(핵심) → CR7(퀄리티) → De Bruyne(사용자) → 승인
```

### 📐 아키텍처 변경 시
```
수비 중심
성민 → Messi(판단) → Cannavaro(구조) → Ramos(리스크) → Modric(태스크) → Coach → 승인
```

### ✍️ 블로그/콘텐츠 작성 시
```
프론트 오피스
성민 → Beckham(작성) → Messi(방향 확인) → 발행
```

## 에이전트 소환 방법

### Claude 대화에서 소환
```
"이 기능에 대해 CR7 관점에서 의견 줘"
"Ramos라면 여기서 어떤 리스크를 볼까?"
"Coach, 지금까지 나온 의견 종합해줘"
```

### 시스템 프롬프트로 소환
```
prompts/ 폴더에서 해당 에이전트의 .md를 가져와서
Claude 프로젝트 지침에 삽입하거나 대화 시작 시 제공
```

### 멀티 에이전트 세션
```
"오늘은 일지 입력 UI를 설계할 거야.
Griezmann, CR7, Son, Modric, De Bruyne, Ramos 투입해줘.
경기 운영법 순서대로 의견 줘."
```
