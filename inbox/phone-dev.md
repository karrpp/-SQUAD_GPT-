# 폰 개발 환경에 대한 나의 경험과 생각

## 시작하게 된 계기

처음 폰으로 개발을 시작한 건 노트북 없이 아이디어가 떠올랐을 때였다.
급하게 뭔가를 고치거나 테스트해야 할 상황에서 폰이 유일한 선택지였고,
그렇게 Termux를 처음 설치하게 됐다.

## 실제로 써보니

솔직히 처음엔 많이 불편했다. 작은 화면에서 코드를 읽는 것 자체가 눈이 아팠고,
터치 키보드로 괄호나 특수문자를 입력하는 게 상당히 번거로웠다.
오타도 잦고, 복잡한 디렉토리 구조를 탐색하는 것도 쉽지 않았다.

그런데 시간이 지나면서 나름의 흐름이 생겼다.
SSH로 원격 서버에 접속하면 폰이 단순한 터미널이 되고,
실제 작업은 서버에서 돌아가니까 성능 문제는 사라졌다.
tmux로 세션을 유지하면 끊겼다 돌아와도 그대로였다.

## 잘 되는 것들

- Git 커밋, 풀, 푸시는 폰에서도 충분히 된다
- 간단한 스크립트 수정이나 설정 파일 편집
- 서버 로그 모니터링과 프로세스 관리
- 마크다운 작성이나 문서 정리

## 한계를 느낀 순간

프론트엔드 작업은 거의 불가능에 가깝다.
브라우저 개발자 도구가 없고, UI를 확인하면서 수정하는 게 너무 힘들다.
대규모 프로젝트에서 여러 파일을 동시에 보며 리팩토링하는 작업도 무리다.
그리고 발열 — 빌드 돌리다 보면 폰이 뜨거워지면서 속도가 확 떨어진다.

## 결론

폰 개발은 "할 수 없다"가 아니라 "어디까지 할 수 있냐"의 문제다.
주 개발 환경이 되기는 어렵지만, 보조 수단으로는 생각보다 훨씬 쓸만하다.
특히 원격 서버와 조합하면 이동 중에도 꽤 많은 걸 처리할 수 있다.
완벽하진 않지만, 없는 것보단 훨씬 낫다.
