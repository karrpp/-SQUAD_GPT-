## 결정 배경

1. **모바일 최적화 앱** — Time Arena는 본질적으로 모바일 환경에서 작동해야 하는 앱. PWA 검증 결과가 모바일 UX를 대변하지 못할 가능성 있음.

2. **경험 획득이 목표** — 수익성/지표 검증보다 "모바일 프로덕트를 끝까지 완성하는 경험"이 이 프로젝트의 진짜 목적. 챔피언스 리그 첫 번째 본선 — 탈락해도 경험이 남는다.

3. **홈페이지는 웜업** — SQUAD GPT 홈페이지(React+Vite)는 상대적으로 쉽게 완성됨. 이제 진짜 도전: 모바일 앱.

---

## 기술 판단 필요 사항 (@Modric)

PWA → 모바일 전환 시 **컴포넌트 공유 문제**가 있음:

| 구분 | PWA (React Web) | React Native |
|------|----------------|--------------|
| UI 컴포넌트 | `div`, `button`, `input` | `View`, `Text`, `TouchableOpacity` |
| 비즈니스 로직 | ✅ 공유 가능 | ✅ 공유 가능 |
| 스타일 | CSS | StyleSheet (다름) |

**선택지 A — PWA 먼저, 이후 React Native 재작성**
- 로직만 공유, UI는 재작성
- 검증 → 전환 2단계 명확히 분리

**선택지 B — Expo (React Native + Web 동시 지원)**
- 처음부터 한 코드베이스로 PWA + iOS + Android
- 학습 비용 있음, 하지만 이중 작업 없음
- 모바일 완성도가 목표라면 더 합리적

→ **@Modric 기술 판단 요청**: 방향이 "PWA 검증 없이 모바일까지 간다"로 확정됐다면, Expo 단일 코드베이스가 더 나은 선택인지 검토 필요.
